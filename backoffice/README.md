# Backoffice Memory

This folder stores reusable logic for admin panels, dashboards, CMS tools, and operational web apps.

Use it when a project needs more than a pretty frontend:
- editable products or services
- staff-only tools
- role-based access
- operational status tracking
- media upload and placement control
- search, filters, or multiple views on the same records

Core references:
- `backoffice/DESIGN-SPEC.md`
- `docs/BACKOFFICE-APP-RESEARCH.md`

Why this exists:
- strong apps are good because the system underneath is clear
- the backend model and the display model need to work together
- future builds should inherit this instead of rebuilding admin logic from scratch
