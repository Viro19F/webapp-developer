# Compliance Memory

This folder stores reusable launch-compliance guidance for websites, especially for EU-facing launches.

Use it when a project needs:
- cookie/consent decisions
- privacy-aware launch planning
- accessibility launch notes
- compliance checklists
- EU-facing website launch guidance

Core references:
- `compliance/LAUNCH-COMPLIANCE-SPEC.md`
- `docs/COMPLIANCE-LAUNCH-RESEARCH.md`

Why this exists:
- compliance is part of launch quality, not only legal cleanup
- future launches should inherit safer defaults automatically
