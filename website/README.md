# Website Systems Memory

This folder stores the reusable operating logic for making, deploying, and selling professional websites.

Use it when a project needs:
- a better build strategy
- stronger performance and accessibility rules
- a deployment recommendation
- a pitch, audit, or proposal
- a repeatable service package instead of a one-off build

Core references:
- `website/BUILD-SPEC.md`
- `website/DEPLOYMENT-PLAYBOOK.md`
- `website/SALES-PLAYBOOK.md`
- `docs/WEBSITE-SYSTEMS-RESEARCH.md`

Why this exists:
- good websites are not only good-looking pages
- they are systems with content, performance, deployment, and commercial logic
- future builds should inherit these patterns automatically
