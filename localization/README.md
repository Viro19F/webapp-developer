# Localization Memory

This folder stores reusable logic for multilingual and multi-regional websites.

Use it when a project needs:
- multiple languages
- country/region-specific pages
- hreflang strategy
- translation architecture
- locale-aware formatting
- international SEO

Core references:
- `localization/LOCALIZATION-SPEC.md`
- `docs/LOCALIZATION-RESEARCH.md`

Why this exists:
- multilingual websites fail when translation, URLs, and regional targeting are treated as an afterthought
- future localized builds should inherit clearer rules automatically
