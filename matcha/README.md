# Matcha Folder

This folder groups the reusable memory from the Mori Matcha experiment.

Use it when a future client is:
- specialty retail
- product-led
- curated instead of catalog-heavy
- physical-first or trust-first
- education-heavy before purchase

## Files

- `DESIGN-SPEC.md` = reusable design system and implementation rules
- `THOUGHT-PROCESS.md` = decision logic behind the page

## Related implementation files

- `clients/mori-matcha/index.html`
- `clients/mori-matcha/shop.html`
- `clients/mori-matcha/product.html`
- `clients/mori-matcha/ritual.html`
- `clients/mori-matcha/contact.html`
- `clients/mori-matcha/styles.css`
- `clients/mori-matcha/products.js`

## How to use this folder

Before building a similar client:
1. Read the client brief
2. Read `INDUSTRY-PLAYBOOK.md`
3. Read `docs/LEARNINGS.md`
4. Read `matcha/DESIGN-SPEC.md`
5. Read `matcha/THOUGHT-PROCESS.md`

The goal is not to copy the page exactly. The goal is to reuse the category logic, conversion strategy, visual discipline, and content structure.
